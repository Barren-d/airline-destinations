# PyFly Revival Plan
_Last updated: 2026-04-17_

## Project Goal
Interactive great circle route visualisation of flight networks, starting with the Spanish AENA
network (scraped live via CLI) and expanding via OpenSky live flight data, with a historical
comparison layer from OpenFlights (2017). Built with Streamlit + pydeck ArcLayer, backed by
DuckDB, with a pluggable multi-source architecture designed for future data providers.

## Source Architecture

### The Three Sources

| Source | File | Data type | Auth | Scope | Role |
|---|---|---|---|---|---|
| **AENA Live** | `aena.py` | Current scheduled routes (scraped) | None | AENA only | Primary — current truth |
| **OpenFlights** | `openflights.py` | Static routes ~2017 | None | All scopes | Historical comparison |
| **OpenSky** | `opensky.py` | Actual flights flown | Required | All scopes | Live operational layer |

**Critical architectural decision:** The AENA scraper (`aena.py`) is a **CLI/cron operation only**.
It writes to DuckDB. The Streamlit app **only reads from DuckDB** — it never triggers a scrape.
This decoupling means:
- App loads are instant (no waiting for playwright)
- Scraper can be scheduled independently (cron, GitHub Actions)
- DuckDB cache protects the app if a scrape fails
- No Playwright dependency in the cloud app environment

### Source Interface (base.py contract)

```python
class FlightSource:
    name: str                          # display name shown in UI toggle
    requires_auth: bool                # whether .env credentials are needed
    supports_scopes: list[Scope]       # which scopes this source can serve

    def fetch(self, scope: Scope) -> polars.DataFrame:
        # must return DataFrame with exactly these columns:
        # origin_iata, origin_lat, origin_lon,
        # dest_iata, dest_lat, dest_lon,
        # airline_iata, airline_name, source
        ...

    def is_available(self) -> bool:
        # returns False if auth is missing or required data files absent
        # used by app.py to disable unavailable sources gracefully
        ...
```

`Scope` is an enum: `AENA | EUROPEAN | GLOBAL_TOP_100 | CUSTOM`

### Standard Output Schema

All sources must return a Polars DataFrame with exactly these columns:

```
origin_iata    str    3-letter IATA code
origin_lat     f64
origin_lon     f64
dest_iata      str    3-letter IATA code
dest_lat       f64
dest_lon       f64
airline_iata   str    2-letter IATA airline code
airline_name   str    full name (e.g. "Vueling")
source         str    source identifier string
```

## DuckDB Cache Strategy

DuckDB is the resilience layer for the entire pipeline. It stores successful results from
all sources and serves the app regardless of whether upstream data is fresh.

```
Table: routes
  origin_iata, origin_lat, origin_lon,
  dest_iata, dest_lat, dest_lon,
  airline_iata, airline_name, source,
  scraped_at TIMESTAMP

Table: cache   (OpenSky per-airport API cache)
  airport_icao, fetched_at TIMESTAMP, data JSON
```

Cache behaviour by source:
- **AENA:** last successful scrape stored indefinitely. If next scrape fails, app serves
  stale data with a banner showing age: _"AENA data from 6 hours ago (refresh failed)"_
- **OpenFlights:** static files, no cache needed — always readable from `data/` directory
- **OpenSky:** per-airport API cache, 24h TTL. On rate limit or timeout, return cached
  data for that airport and continue with others

The app always has something to show. It never fails silently — it shows data age and
source provenance for everything it renders.

### Phase 4 — AENA Scraper Source
**Goal:** working `aena.py` — CLI-triggered, cached in DuckDB, never called from the app

**Status:** `[ ] pending`

**Before writing any code:** complete Phase 2f (anti-bot check). If Cloudflare is detected,
adjust the approach documented here accordingly.

Logic in `pyfly/sources/aena.py`:

1. Load `config/airport_urls.json` — maps IATA → verified destination page URL
2. Use async playwright (headless Chromium) to load the AENA master airport list:
   `https://www.aena.es/es/pasajeros/nuestros-aeropuertos.html`
   — this is the stable index of all AENA airports (confirmed navigation path)
3. Extract each airport's main page link from the master list
4. For each airport: navigate to main page → find "Aerolíneas y destinos" menu →
   follow to "Destinos del aeropuerto"
   - If `airport_urls.json` has an entry for this airport's IATA code, use that URL
     directly instead of navigating (faster, handles known edge cases)
   - Wait for route article elements to render before parsing
   - Reference: `archive/aenadestinations.py` line 65 for original article CSS selector
5. For each airport (1–2s polite delay between requests):
   - Parse rendered HTML with BeautifulSoup
   - Extract destination, country, airlines per route
   - On failure or 0 results: log airport name, continue — do not abort
5. Join with `data/airports.csv` for origin + destination coordinates
6. Join with `data/airlines.dat` for airline full names
7. Write enriched DataFrame to DuckDB `routes` table with `scraped_at = now()`
   - Tag all rows with `source = "aena"`
   - Overwrite previous AENA rows (upsert by source)
8. On completion: print summary — airports scraped, total routes, airports with 0 results

**Caching / resilience:**
- Every successful scrape overwrites the previous AENA data in DuckDB
- If a scrape fails mid-run, partial results are still written (per-airport commits)
- If the scraper fails entirely: DuckDB retains the last successful scrape
- The app reads from DuckDB and shows data age — stale data is shown with a warning,
  not withheld from the user

`supports_scopes = [Scope.AENA]`
`requires_auth = False`

### Phase 5 — OpenSky Source (Live)
**Goal:** working `opensky.py` querying actual flight data with DuckDB cache

**Status:** `[ ] pending`

Logic in `pyfly/sources/opensky.py`:

1. Load credentials from `.env`; `is_available()` returns False if missing
2. Load IATA↔ICAO mapping from `data/airports.csv`
3. Resolve scope to list of ICAO codes
4. For each airport:
   - Check DuckDB cache: `db.check_cache(icao)` — if < 24h old, use cached data
   - On cache miss: `GET /api/flights/departure?airport={ICAO}&begin={ts}&end={ts}`
     (default window: last 7 days)
   - HTTP 429 (rate limit): use stale cache if any exists, skip with warning if not
   - HTTP 401 (bad auth): raise immediately with clear message pointing to `.env`
   - Timeout: retry once after 5s, then skip with warning
   - Write successful response to DuckDB cache with timestamp
5. Aggregate to unique `(origin_iata, dest_iata, airline_iata)` route pairs
6. Join with airports for coordinates, airlines.dat for names
7. Add `source = "opensky"` column

`supports_scopes = [Scope.AENA, Scope.EUROPEAN, Scope.GLOBAL_TOP_100, Scope.CUSTOM]`

### Phase 6 — Error Handling & Resilience
**Goal:** no source failure crashes the app; all failure states surfaced clearly to the user

**Status:** `[ ] pending`

#### AENA scraper
- Per-airport try/except: one bad page does not abort the full scrape
- Playwright timeout: log airport name, skip, continue
- 0-result airport: log — expected for ABC, ODB, HSK etc. (see `url_notes.txt`)
- Total failure (all airports 0): raise `ScraperError` — likely HTML structure changed,
  check `archive/aenadestinations.py` for old selectors to diff against

#### OpenSky
- 429 rate limit: serve stale DuckDB cache + sidebar warning showing cache age
- 401 bad auth: disable OpenSky toggle in UI, show "check .env credentials" message
- Empty response: log airport, skip — some airports have no departure records in window

#### DuckDB
- Corrupt `.ddb` file: delete and recreate on `init_db()` — data will repopulate on next CLI run
- Write failure: log and continue in read-only degraded mode
- Schema mismatch: return empty DataFrame with correct columns, log error

#### Unmatched IATA codes
- Accumulate unmatched codes during enrichment
- After pipeline completes: log count
- If > 5% of rows unmatched: emit visible warning — airports.csv may need refreshing

#### Streamlit surface
- Sources failing `is_available()`: greyed out in toggle with tooltip explaining why
- 0 routes returned: warning banner, not blank map
- Stale AENA data (> 6h): amber banner with `scraped_at` timestamp
- Very stale AENA data (> 48h): red banner — suggests scraper may need attention
- All long operations: wrapped in `st.spinner()` with descriptive message

### Phase 7 — Core Pipeline
**Goal:** `ingest.py`, `enrich.py`, `db.py` wired together — full run via CLI

**Status:** `[ ] pending`

**`db.py`**
- `init_db()` — creates `pyfly.ddb` if absent; if absent, hydrates from any parquet
  snapshots present in `data/` before returning (cold-start strategy for cloud deploy)
- `write_routes(df, source)` — upserts rows tagged by source; after every successful
  write, exports `data/routes_{source}.parquet` (the committed source of truth)
- `read_routes(source=None, scope=None)` → Polars DataFrame
- `get_data_age(source)` → `datetime | None` — when was this source last written?
- `check_cache(airport_icao)` → `(bool, datetime | None)`
- `write_cache(airport_icao, df)`

**Parquet as source of truth:**
After every successful pipeline run, `db.py` exports a parquet snapshot per source:
- `data/routes_aena.parquet`
- `data/routes_openflights.parquet`

These parquet files are committed to git. The `.ddb` is always gitignored.
On startup, if `pyfly.ddb` is missing (fresh clone or Streamlit Cloud), `init_db()`
reads the parquet files and hydrates the DuckDB automatically — no manual step needed.
DuckDB can query parquet natively, so this is a single `COPY` operation.

**`enrich.py`**
- `enrich(routes_df, airports_df, airlines_df)` → enriched DataFrame
- Joins routes ×2 for origin + dest coordinates
- Joins airlines for full name resolution
- Logs unmatched IATA counts
- Returns standard schema DataFrame

**`ingest.py`**
- Source registry: `{"aena": AENASource, "openflights": OpenFlightsSource, "opensky": OpenSkySource}`
- `run(source_name, scope)` → fetch → enrich → write DuckDB
- New source = one line in the registry

**`__main__.py`**
- `python -m pyfly --source aena --scope aena`
- `python -m pyfly --source openflights --scope european`
- `python -m pyfly --source opensky --scope aena`
- Prints row counts and warnings on completion

**Scheduled data refresh via GitHub Actions:**
Add `.github/workflows/scrape.yml`:
```yaml
on:
  schedule:
    - cron: '0 4 * * *'   # 04:00 UTC daily
  workflow_dispatch:         # allow manual trigger

jobs:
  scrape:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
      - run: uv sync
      - run: uv run playwright install chromium --with-deps
      - run: uv run python -m pyfly --source aena --scope aena
      - run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add data/routes_aena.parquet
          git diff --staged --quiet || git commit -m "data: AENA route snapshot $(date -u +%Y-%m-%d)"
          git push
```

Streamlit Cloud pulls the latest commit on each app load — so a fresh parquet commit
automatically flows through to the live app within minutes, with no manual intervention.
OpenSky cache data is ephemeral (Streamlit Cloud restarts clear it) — that is acceptable
since it refetches on demand within the 400 credit/day limit.

## Key Decisions Log

| Decision | Rationale |
|---|---|
| AENA scraper as primary source | routes.dat is ~2017; scraper gives current scheduled routes |
| playwright over requests+bs4 | AENA pages may be JS-rendered; playwright handles this natively |
| Scraper CLI-only, never app-triggered | Decouples heavy scraping from app UX; eliminates Playwright cloud issues |
| airport_urls.json over hardcoded dict | URL overrides are config, not code — no code change needed when AENA restructures |
| OpenFlights as historical comparison | Repositioned from fallback to feature — 2017 data is interesting, not authoritative |
| airlines.dat alongside routes.dat | Resolves 2-letter codes to readable names in tooltips |
| DuckDB as cache and resilience layer | Last successful scrape always available; app never shows a blank on failure |
| Data age indicators in UI | User always knows how fresh the data is; stale ≠ broken |
| Smoke tests before source logic | Each phase has a clear pass/fail criterion; regressions caught immediately |
| st.cache_data on DB reads | Prevents DuckDB re-query on every Streamlit widget interaction |
| Arc density cap (default 500) | Prevents visual noise and GPU strain at global scope |
| is_available() on sources | App gracefully disables unconfigured sources rather than crashing |
| git init in Phase 0 | Full history from the start; prerequisite for Phase 9 deploy |
| Parquet as committed source of truth | `.ddb` is binary and grows; parquet is small, diffable, and lets Streamlit Cloud hydrate DuckDB on cold start without Playwright |
| GitHub Actions scrape workflow | Keeps cloud data fresh daily; scraper runs in CI where playwright is easy to install, not in the Streamlit container where it isn't |
| IATA as key in airport_urls.json | Keyed by IATA code not full name string — resilient to AENA renaming airports |
| nuestros-aeropuertos.html as entry point | More stable master index than deriving airport list from one airport's sidebar nav (2022 approach) |
| Polars over pandas | 10-50x faster, modern API |
| DuckDB over CSV files | Queryable, single file, handles all cache needs |
| Streamlit over Flask | Eliminates frontend build pipeline — pure Python |
| pydeck ArcLayer | GPU-rendered geodesic arcs, built exactly for O/D data |
| uv over pip/venv | Faster, reproducible, modern Python standard |
| OurAirports over OpenFlights .dat | Actively maintained, includes both IATA and ICAO columns |
| IATA as internal key | Consistent across all sources after ICAO→IATA mapping |
